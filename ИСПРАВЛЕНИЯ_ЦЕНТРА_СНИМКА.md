# Исправления проблемы с центром снимка и маркерами

**Дата:** 17 апреля 2026  
**Автор:** Лев 🦁 (ИИ-ассистент)

## Проблемы, которые были исправлены

1. **Несовпадение центра снимка с правильным ответом** - спутниковая карта центрировалась на случайных координатах, а не на реальном центре снимка
2. **Отсутствие визуальных маркеров** - не было желтого крестика на центре спутниковой карты и красного крестика на карте выбора
3. **Невозможность получить центр снимка из API** - не было GET endpoint для получения информации о раунде

## Внесенные изменения

### 1. Backend (game_mock.py)
- **Добавлен новый GET endpoint**: `/api/mock/rounds/{round_id}`
- **Возвращает**: полную информацию о раунде, включая `target_point` (центр снимка)
- **Цель**: frontend может получить реальные координаты центра снимка для отображения

### 2. Frontend (index.js)
- **Добавлена функция `addSatelliteCenterMarker(lon, lat)`**: создает желтый крестик в центре спутниковой карты
- **Добавлен маркер клика на карте выбора**: красный крестик появляется при выборе точки
- **Исправлено центрирование спутниковой карты**: теперь центрируется на реальном `target_point` из API
- **Добавлены глобальные переменные**:
  - `satelliteCenterMarker` - ссылка на маркер центра спутниковой карты
  - `guessClickMarker` - ссылка на маркер клика на карте выбора
- **Обновлена логика следующего раунда**: автоматическое центрирование и добавление желтого крестика для нового раунда

### 3. Визуальные улучшения
- **Желтый крестик** (центр снимка): круг с крестом, хорошо виден на спутниковых снимках
- **Красный крестик** (выбор игрока): иконка галочки в красном круге, появляется при клике на карту выбора
- **Автоматическое центрирование**: спутниковая карта всегда показывает реальный центр снимка

## Как работают исправления

### При старте игры:
1. Frontend получает `round_id` из ответа API
2. Делает GET запрос к `/api/mock/rounds/{round_id}`
3. Получает `target_point` (координаты центра снимка)
4. Центрирует спутниковую карту на этих координатах
5. Добавляет желтый крестик в центр

### При выборе точки на карте:
1. При клике на карту выбора создается красный крестик
2. Координаты сохраняются в `selectedPoint`

### При отправке ответа:
1. Правильный ответ (`correctAnswer`) сравнивается с `target_point`
2. Они должны совпадать (с небольшой погрешностью)
3. Отображаются оба маркера: зеленый (правильный) и синий (выбор игрока)

### При переходе к следующему раунду:
1. Автоматически загружается информация о новом раунде
2. Спутниковая карта центрируется на новом центре снимка
3. Добавляется новый желтый крестик

## Технические детали

### Новый API endpoint:
```http
GET /api/mock/rounds/{round_id}
Response:
{
  "id": "string",
  "target_point": {
    "lat": 51.65805007641265,
    "lon": 120.3465687703515
  },
  ...
}
```

### CSS для маркеров:
- **Желтый крестик**: SVG с кругом и крестом, opacity 0.8
- **Красный крестик**: SVG с красной галочкой в круге

### Координатная система:
- Все координаты в WGS84 (EPSG:4326)
- Преобразование в EPSG:3857 для OpenLayers через `ol.proj.fromLonLat()`
